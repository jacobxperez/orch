# Commercial Licensing for Orch

## Quick Overview

The Orch Runtime Artifact Set is provided for **evaluation use** under the **[Orch Runtime EULA](./RUNTIME-EULA.md)**.

The Runtime Artifact Set includes the sealed Orch runtime binary (`orch/public/wasm/orch*.wasm`) and runtime sidecars, such as fingerprints, manifests, notices, policy bundles, provenance files, attestation files, and integrity files.

If you want to use the Orch Runtime Artifact Set in **production**, **beyond non-commercial evaluation inside a business**, **as part of a hosted service**, **inside an embedded system**, **with an OEM product**, or **integrated into proprietary products or workflows**, you must obtain a commercial license.

If your usage is limited to materials in `orch/**` that are covered by the **[Apache License 2.0](./LICENSE-APACHE.md)**, such as public proxies, SDKs, docs, examples, website materials, or public integration utilities, and you comply with those terms, no commercial license is required.

For a plain-language explanation of Orch’s public distribution model, see **[DISTRIBUTION.md](./DISTRIBUTION.md)**.

---

## When You Need a Commercial License

You must obtain a commercial license if your use of the sealed **Orch Runtime** or Runtime Artifact Set involves any of the following:

- **Production Deployment**  
  Any Orch-powered application or service available to end users, customers, production systems, or production workflows.

- **Internal Business Use**  
  Any use of the sealed Orch Runtime inside an organization, business, company, agency, or institution that is not limited to non-commercial evaluation.

- **Commercial Services**  
  Hosted platforms, SaaS products, PaaS products, cloud services, developer tools, orchestration frameworks, agent platforms, or similar commercial offerings.

- **Integration into Proprietary Systems**  
  Embedding or integrating the sealed Orch Runtime into proprietary runtimes, internal platforms, proprietary products, commercial workflows, or customer-facing systems.

- **OEM / Embedded Use**  
  Bundling, embedding, preinstalling, distributing, or operating Orch inside devices, appliances, packaged software, on-prem deployments, or OEM products.

- **Runtime Hosting, Re-Hosting, or Redistribution**  
  Hosting, mirroring, bundling, re-hosting, publishing, redistributing, or making the sealed Orch Runtime or its sidecars available to third parties.

- **Specialized or Regulated Domains**  
  Use in robotics, blockchain or zk stacks, cryptographic execution layers, embedded hardware, real-time systems, hybrid or quantum orchestration systems, or other specialized commercial environments.

If your usage is only of Apache-licensed public proxies, SDKs, tooling, docs, examples, or integration materials in `orch/**`, and you comply with Apache License 2.0, no commercial license is needed.

If you use the sealed Orch Runtime or Runtime Artifact Set beyond evaluation, commercial licensing is required.

## What’s Included

A commercial license may include, depending on the signed agreement:

- Legal right to use the sealed Orch Runtime in commercial or production environments.
- Permission to integrate Orch into proprietary systems, hosted services, internal platforms, customer-facing applications, or embedded deployments.
- Rights for internal, hosted, embedded, OEM, or third-party deployment models as expressly authorized in the agreement.
- Access to commercial runtime terms that supersede the evaluation-only limits in the Runtime EULA for the licensed use case.
- Optional access to:
  - service-level agreements (SLAs)
  - extended support
  - host compatibility terms
  - attestation or policy-bundle terms
  - OEM, platform, or partner distribution terms

## What’s Not Included

Unless expressly stated in a signed agreement, a commercial license does not allow:

- Resale, transfer, sublicensing, or redistribution of rights granted under the agreement.
- Redistribution, mirroring, re-hosting, or public hosting of the Runtime Artifact Set.
- Publishing the sealed Orch Runtime or its sidecars to package registries, containers, CDNs, app stores, or public artifact repositories.
- Use of Orch trademarks outside attribution, permitted compatibility statements, or approved branding guidelines.
- Use of “Official Orch,” “Certified for Orch,” “Orch-approved,” or similar certification language without written permission.
- Disabling, bypassing, removing, or circumventing runtime attestation, integrity, provenance, capability, policy, observability, or orchestration introspection features.
- Modifying, cloning, repacking, reverse engineering, replacing, or presenting a derived runtime as the official Orch Runtime.
- Any rights not expressly granted in the signed agreement.

> For non-commercial, evaluation-only terms, see the **[Orch Runtime EULA](./RUNTIME-EULA.md)**.

## Relationship to Apache License 2.0

Apache License 2.0 applies only to the Apache-licensed public integration materials in `orch/**`, except the Runtime Artifact Set.

Apache License 2.0 does not grant production, commercial, redistribution, hosting, re-hosting, or modification rights for the Runtime Artifact Set.

Using Apache-licensed Orch SDKs, proxies, docs, examples, or public integration utilities does not by itself require a commercial license.

Using the sealed Orch Runtime or Runtime Artifact Set beyond evaluation does require a commercial license.

## Relationship to the Public Distribution Model

Orch’s public distribution model separates:

- Apache-licensed public integration materials;
- evaluation-only sealed runtime artifacts;
- commercial production runtime rights;
- gated host, OEM, redistribution, and certification rights.

See **[DISTRIBUTION.md](./DISTRIBUTION.md)** for the full plain-language explanation.

This file describes commercial licensing only. It does not replace the Runtime EULA, Apache License 2.0, Trademark Policy, or any signed agreement.
